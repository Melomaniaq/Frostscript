# Introduction
This project aims to make a multi-paradigm, interpreted scripting language for the .NET ecosystem inspired by C# and F#.

It's primary purpose is for learning the lower-level code, techniques, and processes of creating a language.

# Architecture
Frostscript is composed of 4 main components
### [Lexer](/Frostscript/Lexer.cs)
Tokonises the script, converting characters to [tokens](/Frostscript.Domain/Token.cs)
### [Parser](/Frostscript/Parser.cs)
Converts the tokens to an abstract syntax tree
### [Validator](/Frostscript/Validator.cs)
Validates the syntax tree follows language rules, such as type checking and variable scope
### [Interpreter](/Frostscript/Interpreter.cs)
Executes the syntax tree returning the resulting value generated by the script

## Features
Language features are defined in the domain layer. They contain all the parsing, validation and interpreting logic.
The order of operations is defined in [ExpressionTree.cs](/Frostscript.Domain/Features/ExpressionTree.cs)
